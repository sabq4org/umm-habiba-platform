# منصة أم حبيبة التعليمية

## الاسم الرسمي
متوسطة أم حبيبة التعليمية - صبيا

## الاسم التقني
`umm-habiba-platform`

## وصف مختصر
منصة داخلية متكاملة لإدارة العمليات المدرسية في متوسطة أم حبيبة التعليمية بصبيا، تشمل الإدارة، الطالبات، المعلمات، الإداريات، الفصول، الجداول، الدرجات، الشهادات، الإعلانات، التواصل الداخلي، الإشراف التربوي، والتقارير.

## الهدف
توحيد بيانات المدرسة وعملياتها اليومية في نظام واضح وآمن، مع صلاحيات دقيقة لكل فئة مستخدمة، وسجل متابعة للعمليات الحساسة.

## أقسام المشروع
- `01-docs`: وثائق الفكرة والمتطلبات والصلاحيات وخارطة الطريق.
- `02-design`: الهوية البصرية، الألوان، الأفكار، والتصاميم الأولية.
- `03-data-model`: تصور البيانات الأساسية للمنصة.
- `04-modules`: تقسيم الوظائف حسب الوحدات الرئيسية.
- `05-ui-pages`: وصف الصفحات والشاشات.
- `06-security`: الخصوصية، الصلاحيات، النسخ الاحتياطي، وسجل التدقيق.
- `07-reports`: تقارير الطالبات، المعلمات، الفصول، الدرجات، والحضور.
- `08-future-features`: أفكار مستقبلية مثل بوابة ولي الأمر وتطبيق الجوال.

## التشغيل المحلي
بعد تثبيت الحزم، يمكن تشغيل النسخة الأولية بالأوامر التالية:

```bash
npm run dev
```

ثم فتح الرابط الذي يظهر في الطرفية.

## الأمان وسياسات الحساب
- سياسة كلمة المرور: ١٠ أحرف فأكثر، تتضمن حروفاً كبيرة وصغيرة ورقماً ورمزاً خاصاً.
- يُجبَر المستخدم على تغيير كلمة المرور عند أول دخول وعند إعادة تعيينها من قِبَل الإدارة (`/change-password`).
- قفل تلقائي للحساب بعد ٥ محاولات فاشلة لمدة ١٥ دقيقة (يمكن للإدارة فك القفل من إعدادات المستخدمات).
- تحديد معدّل لمحاولات الدخول حسب IP + اسم المستخدم خلال نافذة ١٥ دقيقة (جدول `login_attempts`).
- حماية CSRF لنموذج تسجيل الدخول عبر رمز مزدوج التحقق (Cookie + حقل مخفي) مع التحقق من Origin/Referer.
- جميع نماذج Server Actions تتحقق من الصلاحية عبر `assertPermission`، ومسارات الإدارة محمية بـ `middleware.ts`.

### متغيرات البيئة الأمنية
- `AUTH_SECRET` (إلزامي للإنتاج): سرّ توقيع جلسة المستخدم.
- `AUTH_ALLOWED_ORIGINS` (اختياري): قائمة Origins إضافية مسموح بها مفصولة بفواصل.
- `AUTH_ALLOW_MISSING_ORIGIN=1` (اختياري، غير موصى به): يسمح بطلبات بدون Origin/Referer.

## حالة التنفيذ
